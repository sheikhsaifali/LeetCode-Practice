## 🧠 Approach

This is NOT random problem solving.

Each problem is approached with:
1. **Constraint Analysis**
   - Identify input size → decide time complexity limits
   - Detect patterns (sorted, range, uniqueness)

2. **Pattern Recognition**
   - Two Pointers
   - Sliding Window
   - Binary Search
   - Recursion / Backtracking
   - Dynamic Programming
   - Graphs

3. **Multiple Approaches**
   - Brute force → optimized
   - Compare time vs space trade-offs

4. **Edge Case Thinking**
   - Empty input
   - Duplicates
   - Negative values
   - Large constraints

5. **Code Quality**
   - Clean, readable Java code
   - Optimal complexity
   - Production-level thinking

## ⚡ Complexity Rules (Mental Model)

- `n ≤ 10^3` → O(n²) acceptable  
- `n ≤ 10^4` → borderline  
- `n ≥ 10^5` → prefer O(n) or O(n log n)  
- Always aim for **optimal + scalable solutions**
